: @? = '?'
{
	- @?	: represents a named-subsitution-tag
	- `=` 	: is the euqal symbol character
	- '?'	: single-quotes used for writing any number of unicode character(s) used when matching
}
: Operators are parsed from left to right
{
	- Postfix operators
	{
		- '?'+ 			: expects one or more of the operand
		- '?'* 			: expects zero or more of the operand
		- '?'?			: expects zero or one of the operand
		- '?'{?}		: expects exactly ? of the operand, replaced with n,n₀,n₁,... expects the same number of another curly brace operator that has an identical n,n₀,n₁,...
		- '?'{?,}		: expects exactly ? or more of the operand, replaced with n,n₀,n₁,... expects the same number of another curly brace operator that has an identical n,n₀,n₁,...
		- '?'{?₁, ?₂}	: expects ?₁ to ?₂ of the operand, replaced with n,n₀,n₁,... expects the same number of another curly brace operator that has an identical n,n₀,n₁,...
	}
	- Binary operators
	{
		- '?₁' | '?₂'	: either ?₁ or ?₂ is expected
		- '?₁' ~ '?₂'	: creates a range from the unicode character code point which would match any character that is within that range
	}
	- Special independent symbols
	{
		- `.`	:  matches unicode character
	}
	- Encolusre operators
	{
		- (?₁, ?₂, ?ₙ)	:	Predence operator
	}
	- Idea of how `|` works and any other binary operator
	{
		- 'emeka' 'daniel'|'Johnthan' 	=> 'emeka' ('daniel' | 'Johnthan')
		- ('emeka' 'daniel')|'Johnthan' => ('emeka' 'daniel') | 'Johnthan'
	}
}
